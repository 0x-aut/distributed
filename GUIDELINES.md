# GUIDELINES FOR DISTRIBUTED
---
These are the guidelines for contributing to the project to ensure that people do not write shitty code basically.
---

## GUIDELINES:
- Class names must be in Titles, e.g. class NameOfClass
- Function or method names must be in camelCase, e.g. def nameOfFunction
- Variable names must be seperated by an underscore, e.g. name_of_variable
- Constant names must be seperated by underscore, e.g. NAME_OF_CONSTANT
- Folder names must be together, e.g. generatesql.py
